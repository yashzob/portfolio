Project Overview

Jaipur Saree Project

The Jaipur project is a Django-based e-commerce platform designed to facilitate online shopping for customers. It provides product management, shopping cart, checkout, payment processing, user authentication, and order management functionalities.

Technology Stack

  • Backend: Django 4.2.2 (Python)
  • Database: PostgreSQL
  • Frontend: Django Templates, HTML, CSS, JavaScript
  • Payment Integration: Stripe
  • Communication Integration: Twilio (for WhatsApp and SMS)
  • Other: phonenumber_field for phone number validation

Architecture and Components

The project consists of two main Django apps: "store" and "verification".

  • The store app handles product CRUD operations, shopping cart, checkout, payment, user authentication, and order management.
  • The verification app likely handles user verification processes.
  • The project uses Django's built-in authentication system extended with a custom Customer model.

Features and Functionalities

  • Product Management: Create, read, update, and delete products with version control to handle concurrent updates.
  • Shopping Cart: Session and user-based cart management with AJAX support for adding/removing items.
  • Checkout: Shipping address management, payment method selection (Stripe or Cash on Delivery).
  • Payment Processing: Stripe integration for secure payment handling.
  • User Authentication: Registration, login, logout with email and phone number validation.
  • Order Management: Order confirmation, order history, and transaction tracking.
  • Communication: Twilio integration for WhatsApp and SMS notifications.

Concurrency Control

The project implements optimistic concurrency control on the Product model using a version field.

  • Each product has a version number that increments on every update.
  • When updating a product, the current version is checked against the database version to detect conflicts.
  • If a version conflict is detected (indicating another user has modified the product), the request is pushed into a queue for retry processing using Django Q, ensuring eventual consistency and preventing lost updates.
  • The product update view uses database transactions and select_for_update to ensure atomicity and consistency during updates.
  • This mechanism prevents lost updates and ensures data integrity in concurrent environments.

Database Design

  • Customer model linked to Django User with additional fields.
  • Product model with fields for name, price, stock, image, and versioning.
  • Order and OrderItem models to track customer orders and items.
  • ShippingAddress model to store shipping details.

Security and Session Management

  • Uses Django's security middleware and password validators.
  • Session management for cart and order data.
  • CSRF protection enabled with trusted origins configured.
